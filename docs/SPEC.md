# Survey Data Exploration Agent Spec

## 1. 任务目标

本 spec 用于指导 coding agent 探索一批 survey data，并判断这些数据如何支持 dissertation 中关于性别观念、个体实践结果和代际再生产的研究。agent 的主要任务是围绕研究问题开展数据审计、变量识别、样本构造、描述性分析和探索性建模，形成可复查的分析记录。

本 spec 提供研究问题、优先分析方向和变量建议。agent 可以在清楚记录理由的前提下扩展变量、样本和方法，但每一次扩展都需要说明其与研究问题的关系。

## 2. 总体研究问题

本项目关注三个层级的问题：

1. 性别观念在不同调查数据中如何被测量？
2. 性别观念如何与个体层面的家庭、工作、教育/科学结果相关？
3. 性别观念如何通过父母、子女、同胞结构和家庭资源分配进入代际再生产？

其中，个体实践结果主要回答“观念与行为/结果是否相关”；代际再生产主要回答“父母观念、家庭结构和子女性别如何共同关联子代资源、子代结果和子代观念形成”。

## 3. 数据使用边界

用户会把数据放在一个数据文件夹中。agent 需要递归扫描该文件夹，识别数据文件、代码本、问卷、说明文档和已有处理脚本。

| 数据                 | 使用定位                         | 探索重点                                                     |
| -------------------- | -------------------------------- | ------------------------------------------------------------ |
| CFPS                 | 个体实践结果和代际再生产的主数据 | 2014、2020 性别观念题项；个体、夫妻、亲子、同胞和家庭结构    |
| CGSS                 | 独立重复截面补充                 | 2010、2012、2013、2015、2017、2018、2021、2023 中的性别观念、婚姻、就业、教育等题项 |
| 中国妇女社会地位调查 | 历史截面和女性地位补充           | 1990、2000、2010 中的性别角色、家庭分工、就业、婚育和女性地位变量 |
| CEPS                 | 教育/科学场域的独立补充          | 青少年教育期望、学科兴趣、数学/科学能力自评、家庭教育环境    |
| 省级统计数据         | 地区背景补充                     | GDP、受教育程度、就业率、产业结构、女性劳动参与等            |

数据之间的关系：

1. CFPS 可以做个体、夫妻、亲子、同胞和家庭内部分析。
2. CGSS 和中国妇女社会地位调查可以做独立截面或重复截面分析。
3. CEPS 可以作为教育/科学场域的独立补充。
4. 不同调查没有共同个人 ID 时，不得做个体级匹配。
5. 省级统计数据可以通过地区和年份进入宏观背景或地区控制。

## 4. Agent 的探索任务

### 4.1 数据清单

agent 需要先建立数据清单，记录：

| 项目        | 内容                                             |
| ----------- | ------------------------------------------------ |
| 数据文件    | 文件名、路径、格式、行数、列数、年份/波次        |
| 代码本/问卷 | 文件名、路径、对应年份、是否可读                 |
| ID 变量     | 个人 ID、家庭 ID、夫妻 ID、父母/子女 ID、地区 ID |
| 权重变量    | 个体权重、家庭权重、抽样权重、跨波次权重         |
| 标签信息    | 是否有 variable labels 和 value labels           |
| 地区信息    | 省份、城乡、出生地、现居地、迁移相关字段         |

### 4.2 变量审计

每个候选变量都需要建立审计记录：

| 字段              | 说明                                                         |
| ----------------- | ------------------------------------------------------------ |
| dataset           | 数据集名称                                                   |
| year/wave         | 年份或波次                                                   |
| file              | 原始文件路径                                                 |
| variable          | 变量名                                                       |
| variable_label    | 变量标签                                                     |
| question_text     | 问卷题项原文                                                 |
| value_labels      | 取值标签                                                     |
| conceptual_domain | 性别观念、婚姻、生育、工作、教育、亲子、同胞、地区等         |
| measurement_level | binary, nominal, ordinal, Likert, continuous, count, date/year, ID |
| valid_values      | 有效取值                                                     |
| missing_codes     | 缺失、跳答、不适用、拒答、不知道等编码                       |
| analytic_coding   | 分析中如何编码                                               |
| direction         | 越高越传统、越高越平等、越高越多、无方向等                   |
| notes             | 特殊说明                                                     |

### 4.3 缺失值和标签处理

agent 需要系统检查：

1. 变量标签和取值标签。
2. 缺失值编码、跳答逻辑和不适用编码。
3. 负值缺失码、特殊数值缺失码和文本型缺失。
4. 每个关键变量的有效样本量。
5. 每个分析样本的样本筛选流程。

处理原则：

1. 原始数据保持不变。
2. 缺失码转为分析中的 `NA` 前需要记录原始编码。
3. 二分类变量需要明确处理为 `0/1`，并记录 `1` 的实质含义。
4. 名义类别变量需要作为 factor/dummy/one-hot 或分类模型变量处理。
5. 顺序类别变量需要说明按 ordinal 处理或近似连续处理的理由。
6. Likert 题项构造指数前需要统一方向，并记录反向编码。
7. 职业代码、行业代码、单位性质代码、省份代码等不得直接作为连续变量解释。

## 5. 核心研究问题与建议探索方向

以下问题是优先探索方向。agent 可以根据数据实际情况提出新的变量、分组、样本和分析路径。

### 5.1 性别观念如何被测量？

核心问题：

1. 各调查中有哪些性别观念题项？
2. 这些题项分别测量家庭角色、就业观念、能力信念、领导适合性、教育/科学能力，还是家务平等？
3. 题项方向是否一致？
4. 是否可以构造综合性别观念指数？
5. 是否应保留多个维度或单题分析？

建议输出：

1. 性别观念题项清单。
2. 题项文本和取值标签表。
3. 题项分布表。
4. 方向统一说明。
5. 单题、指数和可能维度的可行性判断。

### 5.2 性别观念与家庭实践结果

核心问题：

1. 不同性别观念的个体在婚姻进入、初婚年龄、初育年龄、生育数量上是否存在差异？
2. 夫妻双方性别观念是否相似？
3. 夫妻观念差异是否与家务分工、婚姻满意度、生育意愿相关？
4. 性别观念是否与家务、照料和亲职实践相关？
5. 这些关联是否随性别、队列、城乡、教育水平和地区而变化？

建议指标：

| 模块     | 候选指标                                                     |
| -------- | ------------------------------------------------------------ |
| 婚姻     | 是否结婚、初婚年龄、婚姻状态、婚姻满意度                     |
| 生育     | 初育年龄、是否生育、子女数、生育意愿、理想子女数             |
| 夫妻匹配 | 夫妻观念差异、夫妻观念绝对距离、夫妻观念组合类型、观念 rank matching |
| 家庭分工 | 家务时间、照料责任、育儿参与、工作家庭冲突                   |

建议方法方向：

1. 对二分类结果使用二分类模型或线性概率模型。
2. 对初婚和初育时点探索事件史或生存分析数据结构是否可行。
3. 对夫妻观念匹配探索差异指标、组合类型和 rank-based matching。
4. 对家务时间、照料时间和收入等连续变量检查分布后选择合适方法。
5. 对生命历程路径可探索 sequence analysis 或 latent class analysis。

### 5.3 性别观念与工作/领导力结果

核心问题：

1. 性别观念是否与劳动参与相关？
2. 性别观念是否与职业类型、单位性质、行业位置相关？
3. 性别观念是否与管理岗位、领导岗位或晋升经历相关？
4. 这些关联是否在男性和女性之间不同？
5. 这些关联是否随教育水平、城乡、队列和地区观念气候变化？

建议指标：

| 模块   | 候选指标                                      |
| ------ | --------------------------------------------- |
| 就业   | 是否有工作、就业状态、工作稳定性              |
| 职业   | 职业类型、行业、单位性质、专业技术岗位        |
| 领导力 | 管理岗位、领导岗位、晋升经历、干部/负责人身份 |
| 收入   | 工资、个人收入、家庭收入、收入 rank           |

建议方法方向：

1. 对就业和管理岗位等二分类结果使用二分类模型或线性概率模型。
2. 对职业类型、行业、单位性质使用分类变量处理。
3. 对收入检查极端值、单位、top coding 和是否需要 log transform。
4. 对性别差异使用性别分组或性别交互。

### 5.4 性别观念与教育/科学结果

核心问题：

1. 性别观念是否与最高教育水平、教育期望、学科兴趣或科学能力自评相关？
2. 性别观念是否与从事科学、技术、专业技术类工作相关？
3. 青少年样本中，家庭性别观念或地区观念气候是否与教育/科学取向相关？
4. 成人最高教育水平往往早于当前观念测量，如何处理时间顺序？

建议指标：

| 模块     | 候选指标                                                 |
| -------- | -------------------------------------------------------- |
| 教育成就 | 最高教育水平、受教育年限、学历                           |
| 教育期望 | 自我教育期望、父母教育期望                               |
| 科学取向 | 数学/科学能力自评、学科兴趣、理工科意向                  |
| 科学工作 | 专业技术职业、科学/技术相关职业、职业编码中的 STEM proxy |

建议方法方向：

1. 成人最高教育水平作为 outcome 时，需要特别说明时间顺序限制。
2. 青少年样本和 CEPS 可用于教育/科学场域的独立补充。
3. 职业编码中的科学/技术工作需要先建立分类规则。
4. 教育和科学结果可按性别、城乡、队列和家庭背景分组描述。

### 5.5 父母性别观念与子代资源分配

核心问题：

1. 父母性别观念是否与子女教育投入相关？
2. 父母性别观念是否与子女家务要求和照料责任相关？
3. 女儿和儿子是否获得不同资源？
4. 这种差异是否随父母观念、同胞结构和地区观念气候变化？

建议指标：

| 模块     | 候选指标                                                     |
| -------- | ------------------------------------------------------------ |
| 父母观念 | 母亲观念、父亲观念、父母平均观念、父母较传统观念、夫妻观念差异、夫妻观念组合 |
| 子代资源 | 教育支出、课外培训、学习监督、父母陪伴、家庭学习环境         |
| 责任分配 | 子女家务时间、照料责任、帮助家庭劳动                         |
| 同胞结构 | Daughter、是否有兄弟、是否有姐妹、兄弟姐妹数、出生序、一子一女样本 |

建议方法方向：

1. 先做亲子匹配样本的描述性分析。
2. 分别探索母亲观念、父亲观念和夫妻组合。
3. 对多子女家庭探索同胞比较。
4. 对一子一女家庭探索家庭内部性别差异。
5. 对家庭内部分析探索家庭固定效应可行性。

### 5.6 父母性别观念与子代结果

核心问题：

1. 父母性别观念是否与子代教育结果相关？
2. 父母性别观念是否与子代心理、自我效能、职业期待或早期就业相关？
3. 这些关联是否对女儿和儿子不同？
4. 子代资源分配是否可以解释部分结果差异？

建议指标：

| 模块         | 候选指标                               |
| ------------ | -------------------------------------- |
| 教育结果     | 学业表现、教育期望、教育成就、升学状态 |
| 心理和能力   | 心理健康、自我效能、学习信心           |
| 职业期待     | 职业期望、专业技术/科学工作期待        |
| 早期成人结果 | 就业状态、教育完成、早期职业位置       |

建议方法方向：

1. 先建立父母观念与子代结果的基础关联。
2. 再加入子女性别和同胞结构。
3. 再探索父母观念与子女性别的交互。
4. 在样本允许时探索家庭固定效应。
5. 把资源分配变量作为机制路径进行描述和探索。

### 5.7 父母性别观念与子代观念形成

核心问题：

1. 父母性别观念是否与 age >= 16 子女性别观念相关？
2. 母亲观念和父亲观念的关联是否不同？
3. 是否存在母女、父子、父女、母子等不同传递路径？
4. 夫妻观念一致或冲突是否与子女观念相关？
5. 地区观念气候是否强化或削弱家庭内部观念传递？

建议指标：

| 模块     | 候选指标                                               |
| -------- | ------------------------------------------------------ |
| 父母观念 | 母亲观念、父亲观念、夫妻观念差异、夫妻观念组合         |
| 子代观念 | age >= 16 子女性别观念题项、家务分工态度、女性就业态度 |
| 传递路径 | 母女、父子、父女、母子分组或交互                       |
| 环境变量 | 省级文本观念气候、省级调查态度、省级统计变量           |

建议方法方向：

1. 先检查父母和子女是否在同一 wave 或相邻 wave 均有观念题项。
2. 分别估计母亲观念和父亲观念的关联。
3. 探索 same-gender transmission 和 cross-gender transmission。
4. 探索夫妻观念组合与子代观念的关系。
5. 报告时间顺序和样本限制。

## 6. 每一次分析的固定输出

每一次具体分析都必须生成一个独立分析记录。无论结果是否显著，均需输出 descriptive table 和方法说明。

建议目录结构：

```text
outputs/survey_exploration/analysis_runs/
  analysis_001_short_name/
    00_question.md
    01_descriptive_table.csv
    02_missing_table.csv
    03_method_note.md
    04_result_table.csv
    05_interpretation_note.md
    scripts/
```

### 6.1 `00_question.md`

必须说明：

1. 本次分析回答哪个研究问题。
2. 使用哪个数据、年份和样本。
3. 核心解释变量、结果变量和分组变量是什么。
4. 这次分析属于个体实践结果、代际再生产、测量探索，还是补充分析。

### 6.2 `01_descriptive_table.csv`

每一次分析都要输出 descriptive table。至少包括：

| 内容         | 要求                                              |
| ------------ | ------------------------------------------------- |
| 样本量       | 总样本量和有效样本量                              |
| 核心变量分布 | 均值、标准差、最小值、最大值，或频数和比例        |
| 分组描述     | 按性别、年龄/队列、城乡、教育、地区或家庭结构分组 |
| 权重说明     | 是否使用权重，使用哪一个权重                      |
| 变量方向     | 尤其是性别观念变量的方向                          |

对于 categorical variables，descriptive table 应报告各类别频数和比例。对于 continuous variables，应报告均值、标准差、中位数、分位数和极端值检查。

### 6.3 `02_missing_table.csv`

必须报告：

1. 每个关键变量的缺失比例。
2. 缺失值原始编码。
3. 缺失处理方式。
4. 样本筛选前后的样本量。
5. 若使用 complete-case sample，需要报告被排除样本的基本特征。

### 6.4 `03_method_note.md`

每一次分析都必须写方法说明。至少包括：

1. 为什么选择这个样本。
2. 为什么选择这些变量。
3. 每个变量如何编码。
4. 二分类变量中 `1` 代表什么。
5. 类别变量如何处理。
6. 性别观念题项是否反向编码。
7. 使用了什么分析方法。
8. 该方法适合回答什么问题。
9. 该方法的解释边界。
10. 哪些替代方法可以在下一步尝试。

### 6.5 `04_result_table.csv`

如果进行模型探索，需要输出结果表。结果表应包含：

1. 模型名称。
2. 样本量。
3. 变量编码说明。
4. 主要系数或统计量。
5. 标准误或置信区间。
6. 权重和固定效应说明。
7. 结果是否只是探索性分析。

### 6.6 `05_interpretation_note.md`

必须说明：

1. 结果支持、部分支持或未支持哪个研究问题。
2. 结果的主要描述性事实是什么。
3. 结果是否可能受到时间顺序、选择性、缺失值或变量测量影响。
4. 下一步需要探索什么。

## 7. Measurement 和编码规范

### 7.1 Binary variables

二分类变量必须编码为 `0/1`，并记录：

```text
1 = 事件发生 / 属于目标类别
0 = 事件未发生 / 不属于目标类别
```

### 7.2 Nominal categorical variables

名义类别变量需要使用 factor specification、dummy variables、one-hot encoding 或分类模型。职业代码、行业代码、单位性质代码和省份代码应先转化为类别变量或理论上有意义的分组。

### 7.3 Ordinal variables

顺序变量可以作为 ordinal variable 处理，也可以在理由充分时近似连续处理。agent 需要记录选择理由，并在关键分析中尝试替代处理方式。

### 7.4 Likert items and ideology scales

性别观念题项通常是 Likert 类变量。agent 需要：

1. 查看题项文本和取值标签。
2. 统一方向，例如 `higher = more traditional`。
3. 对反向题项进行 reverse coding。
4. 报告单题分布。
5. 构造均值指数前检查题项相关、Cronbach's alpha 或 factor structure。
6. 同时保留单题版本、指数版本和因子分数版本用于后续比较。

家务平等类题项常与传统角色题项方向相反，需要单独检查。

### 7.5 Continuous variables

连续变量需要检查单位、极端值、合理范围、top coding、bracket coding、是否需要 log transform 和 winsorization。收入、教育支出、课外培训支出等变量应重点检查单位和极端值。

### 7.6 Date, age, and event time

初婚年龄、初育年龄、就业进入、晋升等时间变量需要检查出生年、事件年、事件年龄、删失状态和是否可构造 person-year 数据。若事件发生在性别观念测量之前，解释时需要记录时间顺序限制。

## 8. 样本构造建议

agent 可以优先构造以下样本，并根据数据实际情况扩展：

| 样本               | 用途                                       |
| ------------------ | ------------------------------------------ |
| 成人个体样本       | 婚姻、就业、教育、收入等                   |
| 未婚风险样本       | 初婚年龄和婚姻进入                         |

## 11. 质量检查清单

在提交任何探索结果前，agent 必须确认：

1. 已读取变量标签和取值标签。
2. 已识别并处理缺失码。
3. 已记录每个关键变量的编码方向。
4. binary variables 已编码为 0/1。
5. nominal categorical variables 已按类别变量处理。
6. ordinal variables 的处理方式有记录。
7. Likert items 的方向已统一。
8. 合成指数已检查题项相关和内部一致性。
9. 每次分析都有 descriptive table。
10. 每次分析都有 method note。
11. 每个模型的分析样本量已报告。
12. 每个模型的样本筛选流程可复现。
13. 不同调查数据没有进行错误的个体级匹配。
14. 原始数据未被修改。
15. 输出中没有泄露个人身份信息。

## 12. 明确禁止的做法

1. 不得把标签编码当作实质连续数值直接使用。
2. 不得把 `-8`, `-9`, `98`, `99`, `999` 等可能的缺失码当作有效值。
3. 不得在未查看 value labels 的情况下给变量命名或解释变量方向。
4. 不得把省份代码、职业代码、行业代码、单位性质代码当作连续变量。
5. 不得把不同调查的个体记录强行合并。
6. 不得把横截面相关解释为因果效应。
7. 不得把性别观念测量之后已经发生的婚育、教育经历解释为观念导致的结果。
8. 不得只报告显著结果；不可行、缺失严重和题项不匹配的结果也必须报告。
9. 不得删除中间审计表和清洗日志。

## 13. 最终交付标准

完成一次合格的数据探索后，用户应能够看到：

1. 当前文件夹中有哪些数据，每个数据能做什么。
2. 每个调查中有哪些可用的性别观念题项。
3. 每个题项的标签、取值、方向和缺失值情况。
4. 哪些变量可以支持个体实践结果分析。
5. 哪些变量可以支持亲子、夫妻、同胞和家庭固定效应分析。
6. 每一次分析对应的 descriptive table 和 method note。
7. 哪些方法可行，哪些方法适合作为补充，哪些方法目前缺少数据支持。
8. 下一步需要用户补充哪些代码本、变量说明或数据文件。
